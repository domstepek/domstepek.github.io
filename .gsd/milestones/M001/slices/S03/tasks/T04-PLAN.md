# T04: Dark Theme and CRT Effect

**Slice:** S03
**Milestone:** M001

## Goal
Dark theme retheme with Space Mono monospace font, CRT scanline overlay, retro pixel cursors, and avatar illustration on homepage

## Must-Haves
- [x] The site uses a dark background (#0a0a0a or similar near-black) with light text across all pages
- [x] All body text renders in a monospace font (Space Mono or similar)
- [x] A subtle CRT scanline overlay is visible across the full viewport
- [x] `src/styles/global.css` exists and provides Dark theme variables, monospace font stack, CRT scanline overlay CSS
- [x] `src/components/layout/BaseLayout.astro` exists and provides Google Fonts link for Space Mono, CRT overlay element

The `pointer-events: none` ensures it never blocks interaction. The `aria-hidden="true"` on the HTML element ensures screen readers ignore it. The low opacity (0.15) keeps text readable. 9. Create retro pixel cursor assets in `public/cursors/`. Generate two small PNG cursor images using inline SVG converted to PNG (or hand-craft them with a script). Both should be white-fill with black pixel outline, classic Mac OS style: **default.png** (32x32): A classic pixel arrow pointer — angled arrow pointing upper-left, with a black 1px pixel border and white fill. Hotspot at (0, 0). **pointer.png** (32x32): A classic pixel hand with index finger pointing up (like the image reference from https://www.figma.com/community/file/939098898751395313). Black pixel border, white fill. Hotspot at (10, 0). Use a Node.js script to generate these pixel-art cursors programmatically. The script should create the PNGs by drawing the pixel patterns onto a canvas and saving them. Run the script, verify the files exist, then delete the script. Alternatively, create them as inline SVGs converted to data URIs if PNG generation is complex — but PNG files in `public/cursors/` are preferred for caching. 13. Add custom cursor CSS rules in `global.css` after the CRT overlay styles: ```css /* --- Retro pixel cursors --- */ *, *::before, *::after { cursor: url('/cursors/default.png'), auto; } a, button, [role="button"], input[type="submit"], input[type="button"], select, summary, label[for], [onclick], [tabindex]:not([tabindex="-1"]) { cursor: url('/cursors/pointer.png') 10 0, pointer; } ``` The fallback values (`auto`, `pointer`) ensure the site remains usable if cursors fail to load. 14. **Add personal avatar to the homepage.** The user will provide the image file at `public/images/avatar.png` before execution. The image is a cartoon illustration (person with glasses and a crab claw) with a white/light background that needs to blend into the dark theme. **In `src/data/home.ts`:** Add an `avatar` field with the image path: ```typescript avatar: "/images/avatar.png", ``` **In `src/components/home/HomePage.astro`:** Add the avatar image in the hero section, positioned similarly to how snwy.me places their anime avatar — above or beside the name/heading. Use an ` ` tag: ```astro <img src={home.avatar} alt="dom" class="home-avatar" width="160" height="160" /> ``` Place it as the first element inside the hero/header area, before the eyebrow text. **In `src/styles/global.css`:** Add avatar styles that make it blend into the dark background. The original image has a white/light background, so apply a CSS treatment to integrate it: ```css /* --- Homepage avatar --- */ .home-avatar { display: block; width: 160px; height: 160px; object-fit: contain; border-radius: 0; mix-blend-mode: lighten; filter: grayscale(0.2) contrast(1.1); margin-block-end: 1.5rem; } ``` The `mix-blend-mode: lighten` on a dark background will make the white background of the image disappear, leaving just the illustration — similar to how snwy.me's avatar feels embedded in the page. If the image has transparency (PNG with alpha), skip the blend mode and just render it directly. Try both approaches and use whichever looks better: - **Option A:** `mix-blend-mode: lighten` (works if image has white bg) - **Option B:** If the PNG already has transparency, just render it normally with no blend mode **CRITICAL CONSTRAINTS:** - Do NOT change any existing text content in home.ts — only ADD the avatar field - Do NOT change any existing HTML structure or data-* attributes in HomePage.astro — only ADD the avatar img element - CSS/visual changes ONLY for the rest of the files - Keep all existing responsive media queries intact (just update any hardcoded colors if present) - Ensure WCAG AA contrast: #e0e0e0 on #0a0a0a is 16.5:1 (passes), #8a8a8a on #0a0a0a is 5.5:1 (passes)
